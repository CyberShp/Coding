## 📌 Slide 2 — 背景与问题

### 当前测试痛点

| 问题分类 | 典型表现 | 根因分析 |
|---------|---------|---------|
| **代码异常分支漏测** | 错误处理路径、资源释放路径未被覆盖，逃逸到现场 | 黑盒测试无法感知代码内部分支结构 |
| **流程并发问题** | 多线程竞态、锁序异常、异步时序偶发性故障 | 缺乏对共享变量和并发路径的系统性分析 |
| **协议报文漏测** | NVMe over TCP / KV 协议的边界值、异常报文场景覆盖不足 | 协议实现与代码流程的关联分析缺位 |
| **测试设计依赖经验** | 测试用例设计质量因人而异，缺乏系统化方法论 | 未建立从代码到用例的结构化分析流程 |

### 核心矛盾

> **黑盒测试不知道代码里哪些函数、哪些故障处理分支会在同一时刻交汇，只能靠 N 次测试提高命中率。**

**讲解要点：**
- 用1-2个真实逃逸缺陷案例说明（如 iSCSI login + 端口闪断 → 控制器下电）
- 强调当前测试手段对"多函数交汇临界点"的盲区
- 引出灰白盒测试的必要性

| **编码阶段** | 代码覆盖率采集配置、增量分析 | 代码提交 | 增量代码覆盖率≥基线 | 覆盖率报告 |
| **DT测试** | 灰白盒用例设计与执行 | 功能测试用例基线化 | 灰白盒用例执行完成 | 灰白盒测试报告 |

- 每个阶段的活动嵌入现有版本交付流程，不额外增加独立阶段
- 出入口标准保证流程可执行、可检查
- 这是项目的"基座"，后续所有方案都在此流程框架下运作

## 📌 Slide 7 — 方案二-A：新增软件特性测试策略

### 适用对象：NVMe over TCP 新增特性

```
设计文档 ──────┐
               ├──→ 协议实现与关键代码流程分析 ──→ DT用例设计与执行
抓包工具 ──────┤     │
               │     ├─ 关键函数调用链梳理
协议标准 ──────┘     ├─ 错误处理分支标记
(NVMe/TCP)           ├─ 并发路径识别
                     └─ 多函数交汇临界点分析
```

**具体做法：**

| 步骤 | 活动 | 工具/方法 | 产出 |
|------|------|---------|------|
| 1 | 协议抓包与报文解析 | Wireshark/tcpdump、协议分析仪 | 协议交互流程图 |
| 2 | 设计文档走读 | 设计文档 + 代码对照 | 关键代码函数清单 |
| 3 | 代码流程分析 | 精准测试工具（覆盖率）+ 代码走读 | 调用链图、分支路径图 |
| 4 | 灰盒用例设计 | 基于调用链 + 多函数交汇点 | 灰盒测试用例集 |
| 5 | 参与DT用例评审 | 灰盒视角补充用例 | DT用例中灰盒用例 |
| 6 | 执行与度量 | 代码覆盖率对比 | 覆盖率提升数据 |

**讲解要点：**
- 测试人员深入代码不是"看懂所有代码"，而是聚焦关键协议流程的实现路径
- 通过抓包建立"协议报文↔代码函数"的映射关系
- 用例设计围绕"多函数交汇临界点"，一条用例验证一个交汇场景

## 📌 Slide 8 — 方案二-B：继承软件特性测试策略

### 适用对象：KV 继承特性

```
SFMEA ─────────┐
               ├──→ 有效测试点提取 ──→ 边界值 & 异常分支覆盖测试
协议规范 ──────┘     │
                     ├─ 历史缺陷模式匹配
                     ├─ 关键边界条件识别
                     └─ 异常注入场景设计
                          │
                          ├──→ 协议分析仪（报文级验证）
                          └──→ 网损仪（网络异常模拟）
```

**具体做法：**

| 步骤 | 活动 | 工具/方法 | 产出 |
|------|------|---------|------|
| 1 | SFMEA分析 | SFMEA表 + 历史缺陷库 | 失效模式与测试点映射 |
| 2 | 协议规范梳理 | 协议标准文档 | 边界值清单、异常场景清单 |
| 3 | 测试点优先级排序 | 风险评分（影响×概率） | 分级测试点清单 |
| 4 | 测试设计与执行 | 协议分析仪 + 网损仪 | 异常报文用例、网络异常用例 |
| 5 | 覆盖率验证 | 精准测试工具 | 代码覆盖率报告 |
| 6 | 缺陷闭环 | 缺陷管理系统 | 新发现问题纳入知识库 |

**讲解要点：**
- 继承特性不是"不测"，而是从SFMEA和协议规范中精准提取测试点
- 利用协议分析仪做报文级精确验证，网损仪模拟真实网络异常
- 每一轮测试的发现反哺SFMEA和知识库

```
│    └──────────┴──────────────┴──────────────────────┘          │
├────────────────────────────────────────────────────────────────┤
│                      精准测试工具层                               │
│    ┌──────────┬──────────────┬──────────────────────┐          │
│    │ 代码覆盖 │  调用链       │  变更影响             │          │

│    └──────────┴──────────────┴──────────────────────┘          │
├────────────────────────────────────────────────────────────────┤
│                      代码 & 业务层                               │
│    NVMe over TCP 代码  │  KV 代码  │  业务流程文档             │
└────────────────────────────────────────────────────────────────┘
```

### 三项核心能力建设

| 能力项 | 输入 | AI处理 | 输出 | 价值 |
|-------|------|-------|------|------|
| **覆盖率AI分析** | 代码覆盖率数据 + 代码结构 | 识别高风险低覆盖区域，关联历史缺陷模式 | 覆盖率风险热力图、补充测试建议 | 从"覆盖率数字"到"风险洞察" |
| **调用链→业务场景** | 代码调用链 + 函数签名 | 调用链语义理解，映射到业务操作序列 | 业务场景描述 + 建议测试步骤 | 自动化"代码→测试"的转换 |
| **异步时序分析** | 并发代码结构 + 时序关系 | 竞态条件识别，时序窗口分析 | 偶发问题触发条件 + 复现建议 | 拦截最难复现的并发类缺陷 |

### GrayScope 工具定位

> **重要说明：** GrayScope 目前作为辅助分析工具处于持续验证和迭代中，能否正式纳入项目工具链取决于后续调试验证结果。

| 维度 | 说明 |
|------|------|
| **当前状态** | 已完成九大分析模块开发（分支路径/边界值/错误路径/调用图/并发风险/差异影响/覆盖率映射/事后分析/缺陷知识库） |
| **项目中的角色** | 辅助验证工具 — 用于对比验证人工分析结果的完整性和准确性 |
| **验证计划** | 在试点特性中选择1-2个关键流程，对比GrayScope分析结果与人工分析结果 |
| **纳入条件** | 分析准确率≥70%，且能显著减少人工分析时间（≥30%） |
| **风险预案** | 即使GrayScope不纳入，方案一和方案二仍可独立运作，不影响项目核心目标 |

## 📌 Slide 12 — 试点特性技术方案

### NVMe over TCP — 关键流程示例

| 关键流程 | 灰白盒测试关注点 | 测试方法 |
|---------|----------------|---------|
| **连接建立（Connect）** | 连接状态机异常分支、超时处理、重连机制 | 代码调用链分析 + 网损仪注入 |
| **命令下发（I/O Submit）** | 命令队列并发、队列满异常处理 | 并发路径分析 + 压力测试 |
| **错误恢复（Error Recovery）** | 错误处理函数交汇、资源释放路径 | 错误路径分析 + 故障注入 |
| **断连处理（Disconnect）** | 断连过程中并发I/O处理、状态清理 | 多函数交汇分析 + 时序测试 |

### KV — 关键流程示例

| 关键流程 | 灰白盒测试关注点 | 测试方法 |
|---------|----------------|---------|
| **KV Put/Get** | 边界键值大小、并发读写竞态 | 边界值分析 + 协议分析仪 |
| **KV Delete** | 删除过程中并发访问、空间回收 | 并发路径分析 + 覆盖率跟踪 |
| **KV List/Iterate** | 迭代过程中数据变更、游标有效性 | 数据流分析 + 异常注入 |
| **异常场景** | 存储空间满、网络闪断、多控切换 | SFMEA + 网损仪 + 故障注入 |
